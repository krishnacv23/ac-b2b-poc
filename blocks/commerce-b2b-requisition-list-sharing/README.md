# Commerce B2B Requisition List Sharing Block

## Overview

The Commerce B2B Requisition List Sharing block handles incoming requisition list share links. It uses the `SharedRequisitionList` container from `@dropins/storefront-requisition-list` to auto-import a shared list into the authenticated buyer's account and redirect them to the list detail page.

If the buyer is not signed in, the block renders the sign-in form inline. After authentication the page reloads with the share token preserved, and the import completes automatically.

## Integration

### URL Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `requisition_id` | `string` | Yes | Share token generated by the sender. If missing, the block redirects to the requisition list grid. |

### Cross-page State

After a successful import, the block stores a pending alert payload in `localStorage` under the key `requisitionListPendingAlert` before navigating to the detail page. The `commerce-b2b-requisition-list-view` block reads and clears this key on mount, then emits a `requisitionList/alert` event so the success alert renders on the detail page.

## Behavior Patterns

### User Interaction Flows

1. **No token** ‚Äî `?requisition_id` is absent ‚Üí redirect to `/customer/requisition-lists`.
2. **Guest** ‚Äî Token present but user is not authenticated ‚Üí render inline sign-in form. `routeRedirectOnSignIn` returns the current URL (token preserved), so after sign-in the page reloads and the import runs automatically.
3. **Authenticated** ‚Äî Token present and user is signed in ‚Üí `SharedRequisitionList` dropin calls the import API, shows a loading spinner, then:
   - **Success** ‚Üí stores pending success alert in `localStorage`, redirects to the list detail page.
   - **Error** (invalid / expired token, API error) ‚Üí dropin renders an inline error alert on the page.

### Authentication

- The auth and requisition-list initializers are loaded as static imports so they are always ready before the dropin renders.
- `checkIsAuthenticated()` guards the authenticated path; unauthenticated users never reach the dropin render call.

## Error Handling

- **Missing token**: Redirected to grid before any dropin code runs.
- **Invalid / expired token**: `SharedRequisitionList` dropin displays an inline error alert (`.shared-requisition-list__container`).
- **`localStorage` write failure**: Caught and silently ignored ‚Äî the redirect still happens; the success alert on the detail page simply won't appear.
- **Network / API errors**: Handled and displayed by the dropin.
